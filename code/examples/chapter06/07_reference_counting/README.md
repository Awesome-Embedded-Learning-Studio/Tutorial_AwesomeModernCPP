# Chapter 6.7: 引用计数的实现与性能

本目录包含引用计数的不同实现示例，重点展示单线程和多线程环境下的区别。

## 文件说明

| 文件 | 描述 |
|------|------|
| `intrusive_refcount.cpp` | 单线程侵入式引用计数（无原子操作） |
| `atomic_refcount.cpp` | 多线程安全引用计数（带原子操作） |

## 关键概念

- **侵入式**: 引用计数作为对象的一部分
- **非原子计数**: 单线程环境，零开销
- **原子计数**: 多线程环境，有性能开销
- **内存序**: relaxed 用于递增，acq_rel 用于递减到零

## 实现选择

| 环境 | 推荐方案 | 理由 |
|------|----------|------|
| 裸机/单核 | 非原子侵入式 | 零开销，足够 |
| 多核/多任务 | 原子侵入式 | 线程安全 |
| ISR 环境 | 避免引用计数 | 确定性要求 |

## 编译运行

```bash
mkdir build && cd build
cmake ..
make
./intrusive_refcount
./atomic_refcount
```
