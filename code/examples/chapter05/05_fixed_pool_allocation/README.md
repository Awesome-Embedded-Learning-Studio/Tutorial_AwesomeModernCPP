# Chapter 5.5 - 固定池分配

本目录包含《固定池分配》章节的代码示例。

## 说明

固定池、Slab和Arena是嵌入式系统中常用的替代堆的内存管理策略，它们提供确定性分配、零碎片和低延迟。

## 示例文件

| 文件 | 说明 | 关键点 |
|------|------|--------|
| `fixed_pool.cpp` | 固定池分配器 | O(1)分配、多尺寸池系统、类型安全包装 |
| `slab_allocator.cpp` | Slab和Arena分配器 | Slab状态管理、Arena线性分配、混合策略 |

## 编译方法

```bash
mkdir build && cd build
cmake ..
make
```

## 运行示例

```bash
./fixed_pool
./slab_allocator
```

## 核心概念

### 固定池（Fixed Pool）

- 固定大小块，O(1)分配/释放
- 使用空闲链表管理
- 不会产生外部碎片
- 适合单一类型对象

### Slab分配器

- 管理固定大小对象池
- 维护三个列表：empty、partial、full
- 优化对象构造/析构
- 适合多种相同类型对象

### Arena/Bump分配器

- 线性分配，仅移动指针
- 极快分配速度
- 不可单独释放
- 适合生命周期一致的对象

### 使用建议

1. **固定池**: 单一类型、频繁分配/释放
2. **Slab**: 多个同类型对象、需要缓存
3. **Arena**: 初始化阶段、临时对象
4. **混合**: 小对象用Slab，大对象用Arena

## 环境要求

- C++17 或更高
- GCC 7+ 或 Clang 5+
