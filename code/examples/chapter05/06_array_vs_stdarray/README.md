# Chapter 5.6 - std::array vs C 数组

本目录包含《std::array vs C 数组》章节的代码示例。

## 说明

`std::array` 是现代C++中替代C数组的推荐方式，它在保持零开销的同时提供了更好的安全性和易用性。

## 示例文件

| 文件 | 说明 | 关键点 |
|------|------|--------|
| `basic_comparison.cpp` | 基础比较 | 声明、初始化、迭代、边界检查、STL集成 |
| `practical_examples.cpp` | 实际应用 | 状态机、环形缓冲区、寄存器映射、查表 |

## 编译方法

```bash
mkdir build && cd build
cmake ..
make
```

## 运行示例

```bash
./basic_comparison
./practical_examples
```

## 核心概念

### std::array vs C数组

| 特性 | C数组 | std::array |
|------|-------|------------|
| 内存布局 | 连续 | 连续，相同 |
| 边界检查 | 无 | `at()`方法 |
| 大小信息 | 函数中丢失 | 保留 |
| 复制 | 需memcpy | 可直接复制 |
| 比较 | 比较地址 | 比较内容 |
| STL集成 | 有限 | 完整支持 |

### 何时使用C数组

1. **启动代码/早期引导**: C++运行时未初始化
2. **链接器段放置**: 需要精确控制内存位置
3. **C API互操作**: 某些特殊C接口
4. **极端资源限制**: 避免任何额外抽象

### 何时使用std::array

1. **业务逻辑代码**: 默认选择
2. **需要边界检查**: 使用`at()`
3. **STL算法使用**: 迭代器支持
4. **作为函数返回值**: 不会退化
5. **编译期计算**: constexpr支持

### 使用建议

1. **优先std::array**: 除非有明确理由使用C数组
2. **使用范围for循环**: 更简洁、安全
3. **利用STL算法**: sort、find等
4. **编译期大小**: 用于模板元编程
5. **C API互操作**: 使用`.data()`获取原始指针

## 环境要求

- C++17 或更高
- GCC 7+ 或 Clang 5+
