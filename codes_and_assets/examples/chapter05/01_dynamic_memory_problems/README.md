# Chapter 5.1 - 动态分配问题

本目录包含《动态分配问题》章节的代码示例。

## 说明

演示嵌入式系统中动态内存分配的代价，包括碎片化、对齐问题和不确定性延迟。

## 示例文件

| 文件 | 说明 | 关键点 |
|------|------|--------|
| `memory_layout_demo.cpp` | 内存布局演示 | 静态区、栈、堆的区别与地址分布 |
| `fragmentation_demo.cpp` | 碎片化演示 | 内部碎片与外部碎片的产生 |
| `alignment_demo.cpp` | 对齐和结构体填充 | 对齐要求、padding、优化技巧 |

## 编译方法

```bash
mkdir build && cd build
cmake ..
make
```

## 运行示例

```bash
./memory_layout_demo
./fragmentation_demo
./alignment_demo
```

## 核心概念

### 内存布局

- **静态区** (.data/.bss/.rodata): 编译期确定大小，程序生命周期存在
- **栈** (stack): 函数调用分配，自动释放，容量有限
- **堆** (heap): 动态分配，灵活但代价明显

### 碎片化

- **内部碎片**: 分配粒度导致的浪费（如16字节粒度分配20字节对象）
- **外部碎片**: 空闲块分散无法合并，导致分配失败

### 对齐

- CPU期望数据按自然边界对齐
- 结构体成员按大小排序可减少padding
- DMA、缓存行需要特殊对齐要求

## 使用建议

1. **优先栈分配**: 小对象、短生命周期
2. **静态预分配**: 可预测大小、长期存在
3. **谨慎使用堆**: 碎片化、不确定性延迟
4. **使用内存池**: 固定大小、确定性分配

## 环境要求

- C++17 或更高
- GCC 7+ 或 Clang 5+
